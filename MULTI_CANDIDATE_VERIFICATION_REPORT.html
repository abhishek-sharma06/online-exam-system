🎯 Concurrency Handling

Multi-Candidate Scenario Flow:

  1. Exam Start: 5+ candidates click "Take Exam" → each gets unique user session
  2. Capture Trigger: Every 13 seconds, JavaScript calls capture functions
  3. User Isolation: Each capture includes session user_id from $_SESSION['user_id']
  4. Unique Filenames: Random component prevents collisions even at same second
  5. Database Record: Capture logged with exam_id + user_id combination
  6. Evidence Gallery: Admin views all captures, filterable by user/exam

Race Condition Prevention:

  • ✓ Timestamp + random component (no timestamp-only filename collisions)
  • ✓ User session isolation (each user has own $_SESSION['user_id'])
  • ✓ Database FOREIGN KEY constraints (referential integrity)
  • ✓ Atomic file write operations (PHP file_put_contents is atomic)
  • ✓ Unique database IDs (auto-increment primary keys)
