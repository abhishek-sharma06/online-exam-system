# Exam System Project — AI Assistant Guidelines

## Project Overview
This is an **online exam proctoring system** built with PHP and MySQL. It monitors candidates, records violations (cheating attempts), and provides admin dashboards for exam management and result analysis.

### Key Features
- **Admin Panel**: Create/manage exams, questions, view analytics, and cheating detection logs
- **Candidate Portal**: Registration, exam taking, webcam/screenshot capture, results viewing
- **Proctoring**: Automatic violation detection (leaving window, rapid clicking, suspicious behavior)
- **Database**: Stores exams, candidates, attempts, violations, and captured evidence
- **Email Verification**: OTP-based candidate authentication (optional)

## Important Coding Standards

### General Rules
1. **Session Management**: Always start with `if (session_status() === PHP_SESSION_NONE) { session_start(); }`
2. **Database Access**: Use PDO only (prepared statements to prevent SQL injection)
3. **Password Hashing**: Use `password_hash($password, PASSWORD_BCRYPT)` and `password_verify()`
4. **Role Checking**: Verify `$_SESSION['role']` before allowing admin/candidate actions
5. **Path References**: Use `$_SERVER['DOCUMENT_ROOT']` for includes

### Authentication Pattern
```php
// Check if user is logged in
if (!isset($_SESSION['user_id'])) {
    header("Location: /exam_system/login.php");
    exit();
}

// Check specific role
if ($_SESSION['role'] !== 'admin') {
    header("HTTP/1.0 403 Forbidden");
    exit("Access Denied");
}
```

### Database Connection Pattern
```php
require_once $_SERVER['DOCUMENT_ROOT'] . '/exam_system/config/database.php';
$database = new Database();
$db = $database->getConnection();

// Use prepared statements
$stmt = $db->prepare("SELECT * FROM users WHERE id = ?");
$stmt->execute([$user_id]);
$result = $stmt->fetch(PDO::FETCH_ASSOC);
```

### Response Format (for API/AJAX endpoints)
```php
header('Content-Type: application/json');
echo json_encode([
    'success' => true/false,
    'message' => 'Error/success message',
    'data' => $data
]);
```

### Inline Comments for Logic Blocks
Every major logic block needs a comment explaining what it does in simple words:

```php
// Fetch all exam attempts for this candidate and sort by latest first
$stmt = $db->prepare("SELECT * FROM candidates WHERE user_id = ? ORDER BY start_time DESC");
$stmt->execute([$user_id]);
$attempts = $stmt->fetchAll(PDO::FETCH_ASSOC);

// Check if any attempt has high-severity violations (possible cheating)
foreach ($attempts as $attempt) {
    $check_violations = $db->prepare("SELECT COUNT(*) as total FROM exam_violations WHERE candidate_id = ? AND severity = 'high'");
    $check_violations->execute([$attempt['id']]);
    $violation_count = $check_violations->fetch(PDO::FETCH_ASSOC)['total'];
    
    // Flag exam as suspicious if violations exceed threshold
    if ($violation_count > 3) {
        $attempt['status'] = 'suspicious';
    }
}
```

## Proctoring System Logic

### Violation Detection (cheating_detector.php)
Monitors and logs:
- **Window Focus**: Candidate leaves exam tab/window
- **Click Patterns**: Rapid or suspicious clicking patterns
- **Tab Switching**: Switching to other browser tabs
- **Webcam/Screenshot**: Captures evidence of violations
- **Mouse Movement**: Unusual activity patterns

### Severity Levels
- `low`: Minor violations (momentary tab switch)
- `medium`: Moderate violations (webcam disabled, multiple window leaves)
- `high`: Critical violations (cheating patterns detected)

### When Adding Violations
Always log with:
```php
$violation = [
    'candidate_id' => $candidate_id,
    'exam_id' => $exam_id,
    'violation_type' => 'specific_type',
    'severity' => 'low|medium|high',
    'evidence_path' => $file_path, // optional
    'timestamp' => date('Y-m-d H:i:s')
];
```

## Common Bugs & Fixes

### Session Lost After Login
- **Cause**: Not calling `session_start()` before `$_SESSION` usage
- **Fix**: Add at top of file: `if (session_status() === PHP_SESSION_NONE) { session_start(); }`

### SQL Injection Risk
- **Cause**: Using `$_GET/$_POST` directly in query
- **Fix**: Always use prepared statements with `?` placeholders and `execute([])`

### Proctoring Not Working
- **Cause**: `proctoring.js` not loaded or `exam.js` conflicts
- **Fix**: Check script loading order in HTML, verify JavaScript console for errors

### Violations Not Logging
- **Cause**: Evidence path doesn't exist or permissions denied
- **Fix**: Verify `assets/uploads/` directories exist, check file permissions (755)

### Admin Can't See Results
- **Cause**: Candidate exam not in "submitted" status
- **Fix**: Check `candidates.status` field, run `verify_system_status.php`

6. **Check file permissions** for upload directories
7. **Verify session management** in every page that uses `$_SESSION`
